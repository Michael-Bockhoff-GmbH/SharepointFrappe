SharepointFrappe Setup Guide

Sync your Frappe file attachments to SharePoint — and keep them in Frappe.

SharePoint Per-DocType rules Templated folders

What SharepointFrappe does

It hooks into Frappe's core file handling, so any attachment can be mirrored to — or stored only in — your cloud drive, based on rules you define per DocType.

🔌

Connect SharePoint

Via an Azure AD app registration.

📂

Per-DocType rules

Frappe only, Cloud only, or Both.

🧭

Templated folders

Build paths from fields, record & doctype names.

🔐

Cached & secure

Tokens cached with expiry; secrets stored encrypted.

1 Install the app

Run these from your bench directory, replacing the placeholders.

cd $PATH_TO_YOUR_BENCH

bench get-app sharepointfrappe $URL_OF_THIS_REPO --branch develop
bench --site $SITE_NAME install-app sharepointfrappe

bench build --app sharepointfrappe
bench --site $SITE_NAME clear-cache
💡
Requirements: Frappe v16+, Python 3.10+.

2 Create a SF Cloud Connection

Open SF Cloud Connection → New and fill in the SharePoint credentials. Hit Connect — a green Connected pill means you're good.

🟦 SharePoint

  1. Register an app in Azure AD and grant Sites.ReadWrite.All.
  2. Copy the Tenant ID, Client ID and create a Client Secret.
  3. Set Site Name — the part after /sites/ in the URL.
  4. Set the Drive Name (the document library to upload into).

3 Add an SF Upload Rule

Open SF Upload Rule → New and choose how a DocType's attachments are stored.

FieldWhat it controls
Target DoctypeWhich DocType's attachments this rule governs.
Storage Mode Frappe only Cloud only Both
SF Cloud ConnectionThe connection to upload through.
Naming StrategyKeep the Original filename or use a Templated one — {name}, {field}.
Conflict BehaviorOn duplicate names: Rename, Replace, or Fail.
Folder SegmentsOne row per folder level, top to bottom.

4 Shape the folder path

Each Folder Segment row is one folder level. Mix and match segment types — the form shows a live path preview as you build it.

Static Text

A literal folder name, e.g. Invoices.

Field Value

A field on the record, e.g. customer.

Record Name

The document's ID.

Doctype Name

The DocType, scrubbed.

Preview: /Invoices /{customer} /{record}
📁
An empty Folder Segments table uploads straight to the connection's root folder.

5 Test it end-to-end

Use the bundled SF Test Upload DocType (or any DocType you wrote a rule for), attach a file, and confirm it lands in the right SharePoint/Drive folder.

✅
Once a rule exists, your team just attaches files the normal Frappe way — SharepointFrappe routes everything automatically. Cloud only files keep working in Frappe because their link points straight at the cloud copy.
